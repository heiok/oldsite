net session 命令详解

网络学堂

在日常职场工作中，常会遇到这样一种情况，如某公司局域网内有很多普通打印机需要共享，而大部份的这些打印机都是安装连接在某部门某位职员的电脑上，之后再由此位职员将所安装好后的打印共享出来，局域网内或同部门的职员要使用此打印机时便连接过去。由于职场职员一般都是使用的XP专业版，而微软的win2000pro或winxp pro对连接数是有限定的，这个数为10。所以当连接到共享打印机职员电脑上的连接数达到10时，第11位用户便会出现无法再连接的情况。出现这种情况，我们就需要用到net session这个命令了，除非你给每位共享打印机的职员都安装服务器版的操作系统吧。


net session在CMD模式下运行，如果输入不带任何参数的net session，将显示连接到本地计算机的全部连接数，输出如下所示：

Computer User name Client type Opens Idle time

\192.168.0.90 ISDR Windows 2000 1 00:00:13
\SHARE Administrator Windows 2000 0 01:05:13

从上面可以看出，有两台计算机连接到本地计算机，且192.168.0.90这台计算机打开了一个共享文件夹，活动时间为00:00:13。

net session语法：

net session [\ComputerName] [/delete]

\ComputerName：表示连接到本地计算机的远程计算机名字，此处也可以是远程主机IP；

/delete：此参数可以结束指定远程主机与本机计算机的连接。

比如，当共享主机的连接数到达10以后，我们可以通过使用net session来查看具体的连接时间，并可以手工删除一些连接，以让有需要的用户再连接进来。

net session \192.168.0.90 /delete 这样就会断开90这台主机与本机的连接，命令输入确定后，会有提示问你是否真的要断开，选择Y。

net session /delete这样会断开所有的连接用户。

net session \SHARE 显示SHARE主机与本机连接的详细信息

注：以上所提到的连接数与网上所说的TCP/IP半开连接数不是同一概念！
