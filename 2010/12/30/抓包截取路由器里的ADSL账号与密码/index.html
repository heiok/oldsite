抓包截取路由器里的ADSL账号与密码

网络基础网络学堂

抓包截取路由器里的ADSL账号与密码

本文只提供大概的思路与步骤，具体详细的实现要靠你们自己去摸索，毕竟经过自己的动手得到的结果，印象总会比看了下教程就做出来要来得深刻！

先说明一些问题

1.为何要抓包这么麻烦？

是的，你当然可以打电话到服务商那里去查询。但有些地区的服务商并不会告知你相关信息，且有些情况是不便明说，或许你懂的我所说的~又或者你就纯当此文是娱乐吧

2.是不是所有的ADSL线路都可以抓到账号密码？

不是的，这与你所在地区的网络结构有很大关系，且与ADSL服务提供商所采取的认证协议有关系，当然如果你是在自己的网络测试，一切由你自己决定！

3.拿到账号密码了可以做什么？

事实上，是没什么用处，因为大部份地区的ADSL都与线路绑定。或许朋友忘记账号密码了，且没有路由器登录密码，你可以通过此法帮他找出来！

实现步骤：

1.建立PPPoE服务器

VM+ROS搭建PPPoE服务器，当然你也可以用其他方法。注意虚拟网卡与实体网卡的桥接设置

ROS安装完成后，建立PPPoE用到的一些命令：

/interface print

/interface pppoe-server server add interface=ether1 service-name=Heiok-PPPoE-Server authentication=pap 建立服务

/interface pppoe-server server print 查看状态

/interface pppoe-server server enable 0 启动服务

/interface pppoe-server server print
2.使用Wireshark获取相关信息

在测试路由器里设置好PPPoE拨号信息，保存，将路由器断电，打开Wireshark监听第一步桥接好的物理网卡，用网线将路由器WAN口与被监听网卡连接起来，路由器通电。

3.查看抓取的数据包

路由器通电后，Wireshark抓包界面会出来很多数据包信息，如果PPPoED协议被发现且找到了第一步建立的PPPoE服务器Heiok-PPPoE-Server，则离成功不完。如果没有则请检查连线。

找到PPP PAP Authenticate-Request这一行就是你要的账号与密码啦！本文只采用PAP认证，当然你也可以尝试一下CHAP认证，就会明白这两种认证的区别了~~~
